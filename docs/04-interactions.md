# インタラクション・状態遷移仕様

## 概要

静的デモサイトでは、バックエンドなしでユーザー操作をシミュレーションする。操作結果は `sessionStorage` に保存し、セッション内で一貫した体験を提供する。ブラウザタブを閉じるとリセットされる。

## 状態管理の仕組み

### SessionStore

```typescript
// store/session.ts

interface DemoState {
  // ログインユーザー
  currentUserId: string | null;

  // デモ中に変更された状態
  mutations: {
    // データセット公開状態の変更
    datasetPublished: Record<string, boolean>;

    // 提出物ステータスの変更
    submissionStatus: Record<string, SubmissionStatus>;

    // 提案ステータスの変更
    proposalStatus: Record<string, ProposalStatus>;

    // 追加されたレビューコメント
    addedReviews: Record<string, ReviewComment[]>;

    // データリクエスト投票
    votedRequests: string[];

    // カタログ編集内容
    catalogEdits: Record<string, Partial<CatalogColumn>[]>;
  };
}

// sessionStorage のキー
const STORAGE_KEY = "syntheticdata-demo-state";

// 読み込み
export function loadState(): DemoState { ... }

// 保存
export function saveState(state: DemoState): void { ... }

// リセット
export function resetState(): void { ... }
```

### データ読み込みフロー

画面でデータを表示する際、以下の順序で処理する:

```
1. モックデータ（静的定数）を読み込む
2. sessionStorage から mutations を読み込む
3. mutations をモックデータにマージする
4. マージ結果を画面に表示する
```

例: 提出物一覧の表示

```typescript
function getSubmissions(): Submission[] {
  const state = loadState();
  return SUBMISSIONS.map(sub => ({
    ...sub,
    status: state.mutations.submissionStatus[sub.submission_id] ?? sub.status,
  }));
}
```

## インタラクション一覧

### 認証系

| 操作 | トリガー | 処理 | フィードバック |
|------|---------|------|-------------|
| ログイン | ログインボタン | currentUserId を sessionStorage に保存 | `/dashboard` へ遷移 |
| ログアウト | ログアウトリンク | currentUserId を null に | `/login` へ遷移 |
| ロール切替 | ログイン画面で別ユーザー選択 | currentUserId を更新 | ナビゲーションが切り替わる |

### HR 操作

| 操作 | トリガー | 処理 | フィードバック |
|------|---------|------|-------------|
| データセット作成 | 作成フォーム送信 | なし（UI のみ） | Toast「デモのため、新規作成はシミュレーションです」 |
| 合成データ生成 | 生成ボタン | 2秒のプログレスアニメーション | 完了後「合成データが生成されました」Toast |
| データセット公開 | 公開ボタン | `datasetPublished[id] = true` 保存 | バッジが「非公開」→「公開中」に変化 |
| カタログ自動生成 | 自動生成ボタン | 1秒のローディング | カタログテーブルにモックデータ表示 |
| カタログ保存 | 保存ボタン | `catalogEdits[id]` に保存 | Toast「カタログを保存しました」 |
| 提出物承認 | 承認ボタン | `submissionStatus[id] = "approved"` | ステータスバッジが「承認済み」に変化 |
| 提出物却下 | 却下ボタン | `submissionStatus[id] = "rejected"` | ステータスバッジが「却下」に変化 |
| 提出物実行 | 実行ボタン | 3秒のローディング | 実行完了表示、結果確認リンク表示 |
| 提案レビュー（承認） | 承認ボタン | `proposalStatus[id] = "approved"` + レビュー追加 | レビュー履歴にコメント追加 |
| 提案レビュー（差戻し） | 差戻しボタン | `proposalStatus[id] = "rejected"` + レビュー追加 | レビュー履歴にコメント追加 |
| 提案コメント | コメントボタン | レビュー追加のみ | レビュー履歴にコメント追加 |

### Proposer 操作

| 操作 | トリガー | 処理 | フィードバック |
|------|---------|------|-------------|
| データセット検索 | 検索バー入力 | クライアントサイドフィルタリング | 一覧がリアルタイム絞り込み |
| データダウンロード | ダウンロードボタン | なし | Toast「デモではダウンロードできません」 |
| プロファイル詳細展開 | テーブル行クリック | アコーディオン開閉 | 統計詳細 + ヒストグラム表示 |
| 提出物作成 | フォーム送信 | なし（UI のみ） | Toast「デモのため、提出はシミュレーションです」 |
| 提案作成 | フォーム送信 | なし（UI のみ） | Toast「デモのため、提案はシミュレーションです」 |
| データリクエスト投票 | 投票ボタン | `votedRequests` に追加、vote_count +1 | 投票数が増加、ボタンが「投票済み」に |
| データリクエスト作成 | 投稿ボタン | なし（UI のみ） | Toast「デモのため、投稿はシミュレーションです」 |

## アニメーション・演出

### プログレスアニメーション（合成データ生成・実行）

```
[=====                    ] 20%  データ読み込み中...
[=============            ] 50%  合成データ生成中...
[====================     ] 80%  品質評価中...
[=========================] 100% 完了!
```

- CSS transition で滑らかに進行
- 各ステップのラベルを段階的に表示
- 完了時にチェックマークアイコンとメッセージ

### Toast 通知

```
┌──────────────────────────────────┐
│ ✓ 合成データが生成されました      │   ← 右上に表示
│                                  │   ← 3秒後に fadeOut
└──────────────────────────────────┘
```

### ローディング表示

- ボタンクリック後、ボタンをスピナーに置換
- 処理中はボタンを無効化（多重クリック防止）

## デモシナリオ（推奨操作手順）

サイトのトップまたはサイドバーに「デモガイド」として表示するウォークスルー。

### シナリオ 1: HR によるデータセット管理

```
Step 1: hr_demo でログイン
Step 2: ダッシュボードで「最近のデータセット」を確認
Step 3: DS0003（非公開）の詳細画面へ
Step 4: 「合成データ生成」を実行（アニメーション）
Step 5: 「公開する」をクリック → 公開状態に変化
Step 6: カタログ編集画面で列定義を確認・編集
```

### シナリオ 2: Proposer によるデータ分析

```
Step 1: user_demo_01 でログイン
Step 2: 「公開データセット」で DS0001 を閲覧
Step 3: データプレビュー・プロファイルを確認
Step 4: 「新規提案を作成」から提案フォームへ
Step 5: 自分の提案一覧で PROP0001（承認済み）の詳細を確認
```

### シナリオ 3: HR による提案レビュー

```
Step 1: hr_demo でログイン
Step 2: 「レビュー待ち提案」から PROP0002 をレビュー
Step 3: レポートとコードを確認
Step 4: コメントを入力して「承認」
Step 5: ステータスが「承認済み」に変化
```

### シナリオ 4: データリクエストと投票

```
Step 1: user_demo_01 でログイン
Step 2: 「データリクエスト」画面へ
Step 3: REQ0001 に投票 → 投票数が増加
Step 4: 新規リクエストフォームを確認
```
