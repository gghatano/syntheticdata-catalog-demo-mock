## 各画面の仕様

### 1. ログイン画面

**パス**: `/login`

**レイアウト**:
- ナビゲーションバーなし
- 中央寄せのログインカード

**要素**:
- アプリタイトル「合成データカタログ」
- ユーザー選択ドロップダウン（テキスト入力ではなく、デモ用にドロップダウンにする）
  - `hr_demo` - HR太郎（データオーナー）
  - `user_demo_01` - 分析一郎（提案者）
  - `user_demo_02` - 分析二郎（提案者）
- 「ログイン」ボタン
- システム説明テキスト（1-2行でコンセプトを説明）

**インタラクション**:
- ユーザー選択 → ログインボタン → sessionStorage にユーザー情報保存 → `/dashboard` へ遷移

---

### 2. ダッシュボード

**パス**: `/dashboard`

**HR ユーザー向け表示**:
- 「最近のデータセット」セクション
  - テーブル: 名前 / 状態（公開中 or 非公開バッジ） / 作成日 / 詳細ボタン
  - 最新5件
- 「レビュー待ち提案」セクション
  - テーブル: 提案ID / タイトル / 提案者 / 提出日 / レビューボタン
  - ステータスが `submitted` の提案のみ

**Proposer ユーザー向け表示**:
- 「公開データセット」セクション
  - テーブル: 名前 / 作成日 / 詳細ボタン
  - 最新5件
- 「自分の提案」セクション
  - テーブル: 提案ID / タイトル / ステータスバッジ / 提出日 / 詳細ボタン
- 「新規提案を作成」ボタン

---

### 3. データセット一覧（HR）

**パス**: `/hr/datasets`

**要素**:
- ページタイトル「データセット管理」
- 「新規作成」ボタン → クリックでモーダル or インラインフォーム表示
  - フォーム: データセット名（テキスト入力）、CSVファイル選択（UI のみ、実処理なし）
  - 「作成」ボタン → Toast通知「デモのため、実際の作成は行われません」
- データセット一覧テーブル
  - 列: データセットID / 名前 / 公開状態バッジ / 作成日 / 詳細ボタン

---

### 4. データセット詳細（HR）

**パス**: `/hr/datasets/:id`

**セクション構成**:

1. **基本情報**
   - データセットID、名前、公開状態バッジ、作成日

2. **ファイル一覧**
   - テーブル: ファイル種別（employee_master / project_allocation / working_hours） / パス / 登録日

3. **カタログ**
   - 列数表示（例: 「12列定義済み」）
   - 「カタログ編集」ボタン → `/hr/datasets/:id/catalog` へ遷移

4. **合成データ**
   - 生成済み合成データテーブル: ファイル種別 / Seed / パス / 作成日
   - 「合成データ生成」ボタン → クリックで生成アニメーション（プログレスバー 2秒） → 完了表示
   - 「公開する」ボタン（未公開の場合） → クリックで公開状態に変更

5. **品質レポート**（存在する場合）
   - サマリーメトリクス（全体スコア）
   - ファイル別メトリクス表示

---

### 5. カタログ編集

**パス**: `/hr/datasets/:id/catalog`

**要素**:
- 「自動生成」ボタン → クリックでローディング表示 → モックカタログデータ表示
- カタログテーブル（編集可能）
  - 列名（読み取り専用）
  - 推定型（読み取り専用）
  - PII フラグ（チェックボックス、トグル可能）
  - 説明（テキスト入力、編集可能）
- 「保存」ボタン → Toast通知

---

### 6. 提出物一覧（HR）

**パス**: `/hr/submissions`

**要素**:
- ページタイトル「提出物管理」
- 提出物テーブル
  - 列: 提出ID / タイトル / 提出者 / データセット / ステータスバッジ / 提出日 / 操作
  - 操作ボタン:
    - ステータスが `submitted` → 「承認」「却下」ボタン
    - ステータスが `approved` → 「合成データで実行」「実データで実行」ボタン
    - ステータスが `executed_*` → 「結果確認」リンク

**インタラクション**:
- 「承認」クリック → ステータスが `approved` に変更、sessionStorage に保存
- 「却下」クリック → ステータスが `rejected` に変更
- 「実行」クリック → ローディング表示（2秒） → 実行完了、結果確認リンク表示

---

### 7. 実行詳細

**パス**: `/hr/executions/:id`

**セクション構成**:
1. **実行情報**: 実行ID、提出物ID、実行モード（合成/実データ）、ステータス、実行日時
2. **標準出力**: コードブロック表示（モックログ）
3. **実行結果**: JSON 表示（モック結果データ）
4. **結果公開設定**: スコープ選択（private / submitter / public）、「公開」ボタン

### 9. 提案レビュー（HR）

**パス**: `/hr/proposals/:id`

**セクション構成**:
1. **提案情報**: 提案ID、データセット名、ステータスバッジ、提出日
2. **概要**: 提案サマリーテキスト
3. **レポート**: Markdown レンダリング表示（react-markdown）
4. **コード**: シンタックスハイライト付きコード表示
5. **レビュー履歴**: 過去のレビューコメント一覧（色分けカード）
6. **レビューフォーム**:
   - コメントテキストエリア
   - 「承認」「差戻し」「コメントのみ」ボタン
   - 操作後 → レビュー履歴に追加、ステータス更新

### 11. データセット詳細（Proposer）

**パス**: `/proposer/datasets/:id`

**セクション構成**:

1. **基本情報**: データセットID、名前、説明、タグ、作成日

2. **合成データ**: ファイル一覧テーブル + ダウンロードボタン（クリックで Toast「デモではダウンロードできません」）

3. **データプレビュー**: 各ファイル種別の先頭10行テーブル表示（タブ切替）

4. **データプロファイル**:
   - プロファイルテーブル: 列名 / 型 / 件数 / NULL数 / ユニーク数 / 統計情報
   - 行クリックで詳細展開（アコーディオン）
     - 詳細統計テーブル
     - ヒストグラム（Chart.js）

5. **品質レポート**: サマリー + ファイル別メトリクス

6. **アクションバー**:
   - 「一覧に戻る」ボタン
   - 「テンプレートダウンロード」ボタン（Toast）
   - 「新規提案を作成」ボタン → `/proposer/proposals/new?dataset=:id`

## 共通 UI コンポーネント

### StatusBadge

ステータスに応じた色付きバッジ。

| ステータス | 色 | ラベル |
|-----------|-----|-------|
| published / approved / succeeded / completed | 緑 | 公開中 / 承認済み / 成功 / 完了 |
| rejected / failed / validation_failed | 赤 | 差戻し / 却下 / 失敗 |
| submitted / under_review / open | 青 | 提出済み / 審査中 / オープン |
| in_progress / running | 黄 | 対応中 / 実行中 |
| draft / unpublished / closed | 灰 | 下書き / 非公開 / クローズ |

### Toast 通知

操作フィードバック用のトースト通知。画面右上に表示、3秒で自動消失。

- **成功**（緑）: 操作完了時
- **情報**（青）: デモ制限の説明時
- **警告**（黄）: 注意事項

### Navbar

ロールに応じたナビゲーションリンクを表示。

**HR ユーザー**:
- ダッシュボード / データセット / 提出物 / 提案 / ログアウト

**Proposer ユーザー**:
- ダッシュボード / データセット / 提出物 / 提案 / データリクエスト / ログアウト
